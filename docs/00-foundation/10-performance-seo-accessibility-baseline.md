# Performance, SEO, and Accessibility Baseline

## Performance

Use React Router SSR, Vercel hosting, TanStack Query caching, route-level behavior, and existing optimization patterns to minimize Firestore reads and rendering cost.

## SEO

Maintain route metadata, `sitemap.xml`, `rss.xml`, canonical behavior, robots output, and content-specific metadata as the public site evolves.

## Accessibility

Use semantic HTML, keyboard-operable controls, labelled inputs, accessible tables, focus order, sufficient contrast, and responsive layouts for public and admin screens.
