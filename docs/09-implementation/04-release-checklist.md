# Release Checklist

## Pre-Release

- `git status --short` reviewed.
- Source changes linted.
- Build checked when runtime or deployment behavior changed.
- Public routes checked signed-out.
- Admin routes checked signed-in where relevant.
- Firestore rules and indexes reviewed for data changes.
- SEO outputs checked for public route changes.
- Documentation updated for architecture, route, rules, or module changes.

## Release Notes

Summarize user-visible changes, security impact, verification performed, and any deferred open decisions.
